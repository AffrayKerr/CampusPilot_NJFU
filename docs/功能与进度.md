# CampusPilot_NJFU 功能与进度说明

本文档总结当前 `CampusPilot_NJFU` 项目已经实现的功能、整体完成度、各模块当前状态、测试覆盖情况以及后续建议。

## 1. 项目定位

`CampusPilot_NJFU` 是面向南京林业大学学生的校园事务辅助系统。当前项目以 Flask 后端、SQLite 数据库、Shell 脚本执行层、Python 爬取/自动化脚本和 Flask 模板前端组成，目标是把校园账号绑定、WebVPN 登录态维护、课表/考试同步、任务提醒、图书馆座位预约、通知、日志和用户反馈整合到一个统一平台中。

## 2. 当前技术架构

```text
前端页面（Flask templates + static JS/CSS）
    ↓ fetch API
Flask API 层（backend/api）
    ↓ 鉴权、校验、用户隔离、统一响应
服务层（backend/services）
    ↓ 数据库访问 / Shell 调用 / 加密 / 通知
Shell 脚本层（shell/*）
    ↓ 业务执行、定时/后台 worker、系统维护
Python 自动化/客户端（schedule_scraper.py、seat_client.py、webvpn_selenium_helper.py）
    ↓
学校系统 / WebVPN / 图书馆座位系统
```

核心特点：

- 本地系统账号与学校校园账号分离。
- 本地账号使用 token/session 登录态。
- 校园网密码加密存储。
- 需要学校系统数据的功能统一要求先绑定校园账号。
- WebVPN 登录采用交互式浏览器方式，适配验证码和复杂跳转。
- 后端 API 不直接写复杂爬虫逻辑，而是调用 Shell/Python 执行层。
- 普通业务数据按 `user_id` 隔离。

## 4. 已实现功能详情

### 4.1 Flask 后端与页面路由

已实现内容：

- Flask 应用创建与配置加载。
- CORS 支持。
- API 蓝图注册：账号、管理员、校园账号、WebVPN、课表、座位、日志、通知、提醒、反馈、用户。
- 页面路由：
  - `/` 登录页
  - `/dashboard` 仪表盘
  - `/schedule` 课表页
  - `/tasks` 任务页
  - `/seat` 座位预约页
  - `/logs` 日志页
  - `/profile` 个人中心
  - `/feedback` 反馈页
- 健康检查接口：`/api/health`

当前进度：后端基础框架已经成型，可继续围绕真实业务稳定性做增强。

### 4.2 本地账号与管理员

已实现内容：

- 普通用户注册。
- 普通用户登录，返回 token、过期时间和用户信息。
- 登录 token 同时支持 Authorization Bearer 和 HttpOnly Cookie。
- 注销登录并撤销 session。
- 查询当前登录用户。
- 修改密码。
- 管理员账号通过 `backend/scripts/create_admin.py` 创建，不允许前端直接注册 admin。
- 注册时根据邮箱初始化通知设置。

当前进度：账号体系较完整。后续建议补充：

- 密码强度策略更细化。
- 登录失败频率限制。
- session 自动清理。
- 管理员操作审计。

### 4.3 校园账号绑定

已实现内容：

- 用户绑定校园账号和校园密码。
- 校园密码加密存储。
- 查询绑定状态。
- 更新校园账号/密码。
- 解绑校园账号。
- 校园账号是 WebVPN、课表同步、考试同步、抢座等功能的前置条件。

当前进度：绑定流程已具备。后续重点是提高前端提示友好度，例如当接口返回 `Campus account is not bound` 时自动引导到个人中心。

### 4.4 WebVPN 登录与会话维护

已实现内容：

- 后端提供 `/api/auth` 系列接口。
- 支持使用已绑定账号触发登录流程。
- 支持交互式浏览器登录：
  - 后端启动脚本。
  - 脚本启动 Selenium/浏览器。
  - 用户在浏览器中完成验证码或统一认证。
  - 自动提取 WebVPN cookie。
  - 写入用户运行时目录和数据库状态。
- 支持查询交互式登录状态。
- 支持检查 session 是否有效。
- 支持刷新 session。
- 支持退出/清理 session。
- 有会话保活设计文档和 `session_keeper.sh`。

当前进度：基本实现，但该模块高度依赖真实学校 WebVPN 页面和验证码策略，属于需要持续联调的模块。

主要风险：

- 学校统一认证页面结构变化。
- WebVPN cookie 有效期变化。
- Selenium/Chrome 环境差异。
- Windows 环境下 Shell、Python、浏览器路径兼容问题。

### 4.5 课表、考试与任务

- 课表同步脚本 `sync_schedule.sh`。

当前进度：功能链路已经搭建，真实教务系统数据抓取是主要联调点。

后续建议：

### 4.6 图书馆座位预约

已实现内容：

- 座位配置增删改查。
- 座位状态查询。
- 立即预约。
- 取消预约。
- 失败重试预约。
- 后台抢座 Worker 启动、停止、状态查询。
- 预约结果查询。
- 支持配置楼层、座位号、优先级、预约日期、预约开始/结束时间、检查开始/结束时间、重试间隔、最大重试次数、最大运行时长、启用状态。
- 支持多预约时间段 `reserve_time_slots`。
- API 层已做时间合法性校验：
  - 图书馆开放时间默认 `07:30`。
  - 周一至周四/周末关闭时间默认 `22:00`。
  - 周五关闭时间默认 `20:00`。
  - 单个预约时间段至少 2 小时。
- `seat_client.py` 实现图书馆系统 API 客户端。
- 已有模块设计文档和参考项目分析文档。
- 已有针对多时间段校验和座位客户端的测试。

当前进度：模块设计和代码框架较完整，进度约 75%。

主要风险：

- 图书馆座位系统 token 获取依赖 WebVPN + 图书馆 SSO 跳转。
- 真实预约接口参数可能随系统变化。
- 多候选座位、并发重试和失败恢复需要真实环境压力验证。
- Worker 在 Windows + Shell 环境下长期运行稳定性需要验证。

后续建议：

- 完善图书馆 token 自动获取和刷新。
- 增加预约前检查当前是否已有预约。
- 增加预约冲突检测。
- 增加 Worker 运行日志在前端实时展示。
- 增加座位库/楼层/房间映射，减少用户手填错误。
