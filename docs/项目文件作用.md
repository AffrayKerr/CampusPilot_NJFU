# CampusPilot_NJFU 项目文件作用说明

本文档用于说明当前 `CampusPilot_NJFU` 项目中各文件/目录的作用。说明范围以项目源码、脚本、模板、数据库结构、测试和文档为主，明确排除 `runtime/users`。同时，`.git`、`.venv`、`.pytest_cache`、`__pycache__`、`.pyc`、运行日志和 SQLite 运行库文件属于生成或运行时产物，不纳入逐项说明。

## 1. 根目录文件

| 文件 | 作用 |
|---|---|
| `.gitignore` | Git 忽略规则，避免提交虚拟环境、缓存、运行时数据、日志等不应入库文件。 |
| `.vscode/launch.json` | VS Code/Cursor 调试配置，用于本地启动或调试后端程序。 |

## 2. 后端 `backend/`

后端基于 Flask，负责页面路由、REST API、账号鉴权、数据库访问、调用 Shell 脚本和统一响应。

### 2.1 后端入口与配置

| 文件 | 作用 |
|---|---|
| `backend/app.py` | Flask 应用入口。创建应用、加载配置、注册所有 API 蓝图，并提供登录页、仪表盘、课表、任务、座位、日志、个人中心、反馈等页面路由，以及 `/api/health` 健康检查接口。 |
| `backend/config.py` | 后端配置文件，集中定义数据库路径、密钥、会话配置、运行目录、Shell 路径等配置项。 |
| `backend/requirements.txt` | Python 依赖清单，用于安装 Flask、Flask-Cors、requests、cryptography、pytest 等后端运行和测试依赖。 |

### 2.2 API 蓝图 `backend/api/`

| 文件 | 作用 |
|---|---|
| `backend/api/__init__.py` | API 包初始化文件，使 `backend/api` 可作为 Python 包导入。 |
| `backend/api/account_api.py` | CampusPilot 本地账号接口。实现注册、登录、注销、当前用户信息、修改密码等功能，并创建/撤销登录会话。 |
| `backend/api/admin_api.py` | 管理员接口。实现管理员权限校验后的反馈管理、系统设置、用户/统计类管理能力。 |
| `backend/api/auth_api.py` | WebVPN 登录态接口。封装登录、交互式绑定、状态查询、刷新、注销等操作，并调用 `shell/auth` 下的脚本。 |
| `backend/api/campus_api.py` | 校园网账号绑定接口。实现校园账号绑定、状态查询、更新、解绑和触发 WebVPN 登录等功能。 |
| `backend/api/feedback_api.py` | 用户反馈接口。支持用户提交反馈、查询自己的反馈，管理员可查看和处理反馈。 |
| `backend/api/log_api.py` | 日志查询接口。提供系统日志/业务日志读取、筛选和返回能力。 |
| `backend/api/notification_api.py` | 通知设置与通知触发接口。管理邮件、桌面通知、抢座结果通知、提醒通知等配置。 |
| `backend/api/reminder_api.py` | 提醒接口。管理课程、考试、任务等提醒规则和默认提醒设置。 |
| `backend/api/schedule_api.py` | 课表/考试/任务接口。负责同步课表、同步考试、查询今日课程、管理任务等能力，主要通过 Shell 脚本实现底层逻辑。 |
| `backend/api/seat_api.py` | 图书馆座位预约接口。负责座位配置增删改查、座位状态查询、立即预约、取消预约、重试预约、抢座 Worker 启停和状态查询；同时包含预约时间合法性校验。 |
| `backend/api/user_api.py` | 用户个人资料与统计接口。提供个人信息、邮箱/通知偏好、用户维度统计等数据。 |

### 2.3 服务层 `backend/services/`

| 文件 | 作用 |
|---|---|
| `backend/services/__init__.py` | 服务层包初始化文件。 |
| `backend/services/auth_service.py` | 后端鉴权服务。实现密码哈希与验证、登录 Session 创建/校验/撤销、从请求中解析 token、`login_required` 和 `campus_account_required` 等装饰器。 |
| `backend/services/crypto_service.py` | 加密服务。用于加密/解密校园网密码等敏感信息，避免明文保存用户凭据。 |
| `backend/services/db.py` | SQLite 数据库访问封装。提供初始化数据库、执行 SQL、查询单条/多条记录等基础方法。 |
| `backend/services/notification_service.py` | 通知服务。封装邮件、桌面通知或业务通知调用逻辑，供 API 或 Shell 调用。 |
| `backend/services/response_helper.py` | 统一响应封装。提供 `success_response`、`error_response` 等方法，保证 API 返回结构一致。 |
| `backend/services/shell_runner.py` | Shell 调用封装。负责定位脚本、设置运行环境、传递参数、设置超时、解析脚本 JSON 输出并返回给 Flask API。 |

### 2.4 工具层 `backend/utils/`

| 文件 | 作用 |
|---|---|
| `backend/utils/__init__.py` | 工具包初始化文件。 |
| `backend/utils/validators.py` | 参数校验工具。包含必填字段检查、用户名/密码校验、布尔值归一化等通用校验逻辑。 |

### 2.5 后端脚本 `backend/scripts/`

| 文件 | 作用 |
|---|---|
| `backend/scripts/create_admin.py` | 创建管理员账号的命令行脚本。管理员不能通过前端注册，需使用该脚本写入数据库。 |

## 3. 数据库 `database/`

| 文件 | 作用 |

| `database/migrations/004_add_feedback.sql` | 增加用户反馈相关表，用于问题反馈、建议和处理记录。 |
| `database/migrations/005_add_account_auth.sql` | 增加本地账号认证相关结构，如用户 Session、校园账号绑定等。 |

| `database/migrations/008_add_default_reminder_settings.sql` | 为通知/提醒设置增加默认提醒规则，如课程、考试、任务的默认提前提醒时间。 |

> 说明：`database/campus_pilot.db`、`database/campuspilot.db` 以及类似异常命名的 `.db` 文件是 SQLite 运行数据库，不属于源码文件。

## 5. Shell 公共模块 `shell/common/`

Shell 层承接后端 API 的业务执行，公共模块提供环境、数据库、日志、网络、Cookie 和响应封装。

| 文件 | 作用 |
|---|---|
| `shell/common/env.sh` | Shell 运行环境初始化。定义项目根目录、数据库路径、运行目录、日志路径等环境变量。 |
| `shell/common/db.sh` | SQLite 操作封装。提供执行 SQL、查询数据、事务或常用数据库函数。 |
| `shell/common/log.sh` | 日志封装。负责向日志文件或数据库写入模块日志、错误日志和调试信息。 |
| `shell/common/response.sh` | Shell JSON 响应封装。统一输出 `success/message/data` 格式，便于后端解析。 |
| `shell/common/network.sh` | 网络请求辅助函数。封装 curl 请求、超时、重试、状态码处理等网络能力。 |
| `shell/common/cookie.sh` | Cookie 文件管理。负责读取、保存、检查 WebVPN 或业务系统 cookie。 |

## 6. WebVPN/认证脚本 `shell/auth/`

该目录负责校园网/WebVPN 登录态管理。由于学校统一认证存在验证码/交互限制，当前项目以真实浏览器交互式登录为主。

| 文件 | 作用 |
|---|---|
| `shell/auth/runtime.sh` | 用户运行时路径管理。根据用户 ID 创建/定位运行目录、cookie 文件、状态文件、日志文件等。 |
| `shell/auth/save_credentials.sh` | 保存校园网账号凭据的辅助脚本，配合后端加密存储或运行时使用。 |
| `shell/auth/login_bound.sh` | 使用已绑定校园网账号触发 WebVPN 登录流程的脚本入口。 |
| `shell/auth/bind_webvpn_interactive.sh` | 交互式 WebVPN 登录脚本。启动浏览器/自动化助手，让用户在浏览器中完成验证码或统一认证登录，并保存 cookie。 |
| `shell/auth/webvpn_selenium_helper.py` | Selenium 浏览器自动化辅助程序。负责打开 WebVPN、等待用户完成登录、提取 cookie、写入状态和日志。 |
| `shell/auth/webvpn_client.py` | WebVPN 客户端辅助程序。负责通过 requests/cookie 检查登录态或访问 WebVPN 资源。 |
| `shell/auth/check_session.sh` | 检查 WebVPN 登录态是否有效，更新数据库中的 session 状态。 |
| `shell/auth/refresh_session.sh` | 刷新 WebVPN 登录态或尝试延长当前会话可用时间。 |
| `shell/auth/session_keeper.sh` | 会话保活脚本。周期性检查/刷新 WebVPN 登录态，减少 cookie 失效对课表同步和抢座的影响。 |
| `shell/auth/logout.sh` | 注销 WebVPN 登录态，清理 cookie 或更新会话状态。 |

## 8. 图书馆座位脚本 `shell/seat/`

该目录负责图书馆座位预约模块。整体流程为：前端配置 → Flask API 校验 → Shell 管理配置/Worker → `seat_client.py` 请求图书馆系统 API。

| 文件 | 作用 |
|---|---|
| `shell/seat/seat_client.py` | 图书馆座位系统核心客户端。负责通过 WebVPN cookie 和图书馆 token 调用座位查询、预约、取消、结果记录等 API。 |
| `shell/seat/seat_config.sh` | 新增座位预约配置，写入 `seat_configs` 表。 |
| `shell/seat/list_configs.sh` | 查询当前用户的座位预约配置列表。 |
| `shell/seat/update_config.sh` | 更新已有座位预约配置，包括座位号、时间段、重试策略、启用状态等。 |
| `shell/seat/delete_config.sh` | 删除座位预约配置。 |
| `shell/seat/check_seat.sh` | 查询指定座位或楼层的可用状态。 |
| `shell/seat/reserve_seat.sh` | 执行一次座位预约操作，可用于立即预约。 |
| `shell/seat/cancel_seat.sh` | 取消当前或指定的座位预约。 |
| `shell/seat/retry_seat.sh` | 按配置进行重试预约，适用于候选座位轮询和失败重试。 |
| `shell/seat/list_results.sh` | 查询座位预约结果记录，包括成功、失败及失败原因。 |
| `shell/seat/seat_worker.sh` | 后台抢座 Worker 主循环。按配置检查时间、查询座位并尝试预约。 |
| `shell/seat/start_worker.sh` | 启动当前用户的抢座 Worker。 |
| `shell/seat/stop_worker.sh` | 停止当前用户的抢座 Worker。 |
| `shell/seat/worker_status.sh` | 查询抢座 Worker 是否运行、进程信息和状态。 |
